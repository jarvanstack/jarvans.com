# 不要只关心需求单

我见过很多开发很久但是没人用的功能, 后面细细想，这些功能不仅在浪费团队的资源，还在浪费我的时间，感到难受。

怎么办呢？我的建议是:

**不要只关心需求单**

**因为完成需求不等于创造价值。** 

那什么才叫创造价值呢？

**对用户有价值, 才算创造了价值。**

这是显而易见的事情，但不明白很少人这样思考。

> 药剂师和医生

比如一个你是一个家长，你家孩子发烧了。

你对医生说：“你给我家孩子输液吧，隔壁家孩子也这样，输两天液就好了。”

你是希望:

1. 医生直接给你家孩子输液
2. 还是医生详细询问了你家孩子的病症，说这个只是普通的病毒性感冒，输液会导致抗生素耐药性。回家多休息或开点退烧药就行了。

显而易见，大多数人都希望是后者。

但程序员往往会选择前者，不管产品想要什么功能，程序员直接交付功能。
