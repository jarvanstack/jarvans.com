# 程序员 10x 工作法

> 我是 Jarvan, 这个文档目标是解决 2 个问题:

* 什么是好的程序员?  
  * 招聘角度: 专业能力 VS 价值观, 哪个更重要?
  * 工程师的"五个等级", 你处在哪一级?
* 怎么成为一个好的程序员? 专业能力, 价值
  * 什么是好的程序员? - trade-off 权衡质量与速度
  * 如何性能好? - 封底估算
  * 如何可维护性好? - 设计原则和品味
  * 如何开发快? - 任务分解
  * 如何bug少? - 别把 QA 当啄木鸟
  * 马斯克 5 步工作法
  * 任务分解
* 怎么成为一个好的程序员? 价值观
  * 你认为最重要的 3 个价值观是什么? 坦诚, 积极, 渴望
  * 坦诚 - 红线
  * 主动 - 你由你自己主动控制, 而不是别人
  * 
* 不为清单 - 专业能力
  * 不封底估算, 交白卷
  * 不做 Checklist, 交白卷
  * 不做 40 分的创新
* 不为清单 - 价值观
  * 不撒谎, 不懂装懂
  * 不


```bash
  * 质疑: 不要拿到需求就开始做 - 先质疑所有需求
  * 删除: 不要默认需要完成 100% 的功能 - 先删除非核心的功能
  * 优化: 不要应该只有一个方案 - 你应该有多个方案选择最优
  * 自动化: 不要一开始就做自动化 - 自动化是最后一步
```



> 培训的成本很贵，但不培训的成本更贵。不知道第二点的团队走不远
