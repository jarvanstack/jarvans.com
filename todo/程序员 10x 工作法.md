# 程序员 10x 工作法

> 我是 Jarvan, 这个文档目标是解决 2 个问题:

* 什么是好的程序员?  
  * 招聘角度: 专业能力 VS 价值观, 哪个更重要?
  * 工程师的"五个等级", 你处在哪一级?
  * trade-off 权衡质量与速度
  * 以终为始, 思考如何带来价值, 而不是埋头做需求
* 怎么成为一个好的程序员? 专业能力
  * 马斯克 5 步工作法
    * 拿到需求第一步: 质疑, 默认任何需求都不需要做
    * 第二步: 删除, 不要默认需要完成 100% 的功能
    * 第三步: 优化, 不要应该只有一个方案 - 你应该有多个方案选择最优
  * 如何性能好? - 封底估算
  * 如何可维护性好? - 设计原则和品味
  * 如何开发快? - 任务分解
  * 如何bug少? - 别把 QA 当啄木鸟
  * 任务分解
  * 代码信心从哪里来?
  * 完成的定义, 不是开发完成就叫完成
  * 怎么决定事情优先级?
  * 什么时候提问, 寻求帮助?
  * 如何正确的提问?
  * 如何开会？
* 怎么成为一个好的程序员? 价值观
  * 你认为最重要的 3 个价值观是什么? 坦诚, 积极, 渴望
  * 坦诚 - 红线
  * 主动 - 你由你自己主动控制, 而不是别人
  * 渴望: 投资人, 稻盛和夫, 脑科学
* 不为清单 - 专业能力
  * 不封底估算, 不交白卷
  * 不做 Checklist, 不交白卷
  * 不做 40 分的创新
* 不为清单 - 价值观
  * 别撒谎, 不懂装懂
  * Ego 别太大: 
  * 别受害者心态
  * 别装一团和气
* 阅读书籍
  * 非暴力沟通
  * 高效能人士的七个习惯
  * 代码简洁之道-程序员的自我修养


```bash
  * 质疑: 不要拿到需求就开始做 - 先质疑所有需求
  * 删除: 不要默认需要完成 100% 的功能 - 先删除非核心的功能
  * 优化: 不要应该只有一个方案 - 你应该有多个方案选择最优
  * 自动化: 不要一开始就做自动化 - 自动化是最后一步
```
