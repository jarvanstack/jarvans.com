# 程序员，不要只关心需求单

有个孩子发烧了，家长带他去医院。

家长说：“隔壁家孩子上次也是这样，输两天液就好了。你也给我家孩子输液吧。”

医生没有马上开药，而是先检查。最后发现只是普通的病毒性感冒，回家休息、必要时吃退烧药就行，输液反而没有用。

家长提出的是一个方案，医生关心的却是孩子真正的问题。

**专业，不是别人要什么就给什么，而是判断什么对他真正有用。**

程序员也一样。

产品经理说要做一个 Excel 导出，不要马上问导出哪些字段。先问：谁要用？用它解决什么问题？有没有更简单的办法？

也许问完以后，导出确实要做；也许用户只是想复制二十个订单号，一个复制按钮就够了。

需求单只是别人先想到的答案，不是用户的问题本身。

**完成需求不等于创造价值。用户的问题被解决，才算真正完成。**
