# 别撒谎，别不懂装懂

> 核心观点：承认不知道只产生一次信息缺口，伪装知道会让所有后续决策建立在错误信息上。

## 建议结构

1. 写一次不懂装懂如何误导团队。
2. 分清“不知道”“不确定”和“没有验证”。
3. 给出专业表达：当前结论、依据、未知项、验证时间。
4. 说明负责人如何回应“我不知道”，让坦诚成为理性选择。
5. 讨论紧急情况下必须先行动时如何暴露风险。

## 可用表达

```text
这部分我现在不知道。已确认的是……，尚未确认的是……。
我准备通过……验证，预计在……之前更新结论。
```
