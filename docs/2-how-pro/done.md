# 什么才算完成？

> 核心观点：代码写完不等于任务完成；只有价值被验证、风险被控制、责任被交接，工作才真正完成。

## 建议结构

1. 写一次状态显示“完成”但实际无法使用的任务。
2. 区分编码完成、开发完成、发布完成和结果完成。
3. 给出不同风险级别的完成标准。
4. 说明文档、监控、灰度、回滚和复盘由谁负责。
5. 讨论长期指标需要时间观察时，任务如何关闭。

## 完成清单

- [ ] 验收条件通过
- [ ] 自动化测试和必要的人工验证通过
- [ ] 监控、告警和回滚方案就绪
- [ ] 文档及相关人员已经同步
- [ ] 上线结果符合预期，遗留项已有负责人

请用真实项目调整这份清单，不要把清单本身当作完成。
