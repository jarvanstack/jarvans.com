# 拿到需求，你第一件事情是做什么？

先讲一个故事。

有个孩子感冒发烧，家长带他去医院。

家长说：“隔壁家孩子上次也是这样，输两天液就好了。你也给我家孩子输液吧，明天还要上学。”

医生检查后说：“每个孩子的情况不一样。你家孩子只是普通感冒，回家休息、多喝水就行，不需要输液。”

家长说：“钱我出，你开就好了。”

医生回答：“这不是钱的问题。孩子不需要的治疗，我不能给他开。”

**专业，不是别人做了什么，我们就跟着做什么，而是判断自己究竟需不需要。**


有个小孩发烧了，家长带他去医院。家长说：“给我家小孩输液吧，明天还要上学。上次隔壁家孩子也这样，输两天液就好。”

医生没有给他输液，先检查。检查完说：“你家孩子只是普通感冒，回家休息一下就行，不需要输液。”

家长说。

程序员的目标应该是给用户带来价值，而不是完成一个单子。


对做软件的人来说，我们应该把“终”定位成做一个对用户有价值的软件，能够为别人带来价值，自己的价值才能体现出来。



* 上策: 用户需求得到满足, 产生价值
* 中策: 正确实现产品想要的功能
* 下策: 拿到需求开始干, 没花时间理解需求是什么
