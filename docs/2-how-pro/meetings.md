# 如何开会？

> 核心观点：会议应该用来同步分歧和做决定，单向传递信息通常不需要开会。

## 建议结构

1. 从一次没有结论的会议写起。
2. 会前明确目标：决策、评审、共创还是同步。
3. 只邀请对决策有必要的人，并提前发送材料。
4. 会中围绕分歧和证据推进，记录决定而非逐字记录。
5. 会后明确负责人、截止时间和未决问题。

## 会议模板

```text
目标：本次必须得到什么结果
输入：参会前需要阅读的材料
议题：需要讨论的分歧
决定：选择了什么，为什么
行动：负责人、截止时间、完成标准
```
