# 决定优先级

> 核心观点：优先级不是声音大小，而是价值、紧迫度、成本和风险的共同结果。

## 建议结构

1. 描述多个任务同时“最高优先级”的场景。
2. 先明确共同目标，再比较任务对目标的贡献。
3. 用价值、时间窗口、工作量、风险四项排序。
4. 说明依赖关系和切换成本如何改变顺序。
5. 写清楚“不做什么”以及重新评估的时间。
