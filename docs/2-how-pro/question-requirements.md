# 质疑需求

> 核心观点：接到需求的第一步不是开始实现，而是验证问题和约束是否真实。

## 建议结构

1. 写一次“正确实现了错误需求”的经历。
2. 解释质疑不是反对，而是共同校准目标。
3. 给出追问顺序：目标、用户、证据、约束、成功标准。
4. 展示如何用小实验验证需求。
5. 说明时间很紧时至少要问哪三个问题。

## 待补材料

- [ ] 原始需求原话
- [ ] 追问后发现的错误假设
- [ ] 修改前后的范围与收益
- [ ] 一段不冒犯人的对话示例

## 开工前先问

- 谁提出的要求，他掌握了什么信息？
- 不做这件事的具体后果是什么？
- 什么结果能证明需求有效？
