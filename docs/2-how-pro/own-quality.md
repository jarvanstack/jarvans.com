# 别把 QA 当啄木鸟

> 核心观点：开发者对交付质量负第一责任，QA 的价值是发现系统性风险，而不是重复寻找低级错误。

## 建议结构

1. 写一次测试阶段被大量低级问题阻塞的经历。
2. 区分开发自测、自动化测试和探索性测试的职责。
3. 给出提测前必须完成的最小证据。
4. 说明缺陷逃逸后如何改进系统，而不是只追责个人。
5. 讨论没有专职 QA 的团队如何守住质量。

## 提测证据

- 核心路径已经自测，并记录结果。
- 主要异常路径和边界条件已经覆盖。
- 影响范围、回滚方式和已知风险已经说明。

## 待补材料

- [ ] 一个缺陷从产生到发现的完整过程
- [ ] 哪道防线本应更早发现它
- [ ] 改进后的缺陷率或返工时间
