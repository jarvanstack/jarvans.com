# 代码信心从哪里来？

> 核心观点：对代码的信心来自可重复的证据，而不是“我看过了，应该没问题”。

## 证据层次

1. 问题和预期行为有明确描述。
2. 小范围测试证明核心逻辑正确。
3. 集成测试证明模块能正确协作。
4. 代码评审验证遗漏的假设和风险。
5. 灰度、监控和回滚控制线上不确定性。

## 建议写法

选择一次高风险改动，按提交前、合并前、上线前、上线后四个阶段说明你收集了什么证据。写清每种证据能证明什么、不能证明什么。
