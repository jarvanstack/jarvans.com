# 完成的定义

完成的定义

我见过很多程序员都遇到这个问题（包括我自己）

比如产品问什么时候能开发完，程序员说 两天后。

两天后 产品问为什么还没上线 程序员说两天开发完还没测完呢

为什么会这样呢？因为两个人对完成的理解是不一样的

**程序员认为开发完成叫做完成，产品认为上线叫做完成**

| 状态                             | 分数   |
| -------------------------------- | ------ |
| 开发完成, 没有单元测试           | 不及格 |
| 开发完成, 有单元测试             | 不及格 |
| 测试通过, 没有观察日志和数据     | 不及格 |
| 测试通过, 有观察日志和数据       | 不及格 |
| 上线完成, 没有 checklist         | 不及格 |
| 上线完成, 有 checklist           | 60 分  |
| 上线并观察用户问题是否真的解决了 | 90 分  |


> checklist 是一系列的检查清单, 一座高楼能够没有问题地建出来，全靠检查清单。
