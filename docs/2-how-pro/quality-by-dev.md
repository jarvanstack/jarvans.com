# 代码质量由谁负责?

## 1. 背景

我们团队之前的代码质量是由 QA 负责的，假如线上出 P0 级 bug，QA 要承担责任。

QA 就做事越来越谨小慎微，测试时间越来越长，产品迭代变慢，但其实线上的 bug 也没有明显减少。

所以我就在想，这是怎么回事呢？

## 2. 谁应该负责?

首先明确第一个原则，负责人只能有一个。

所以要么是开发，要么是测试。

很明显不能是测试，因为测试根本不知道开发改了什么，可能一个跟版本完全无关、十万八千里的功能会突然坏掉。

可能开发也不知道这个事情，但是离真相最近的人只能是开发。

## 3. 开发流程

下面是我们团队现在的开发流程。

* 代码质量由开发负责，QA 为辅助角色。
* 如果代码信心足够，可以直接跨过 QA 直接上线，这个时候迭代会非常快。
* 如果代码信心不足，需要他人协助，这个时候迭代会变慢。


![](https://cdn.jarvans.com/blog/202320260720223719639.png)


## 4. 代码信心从何而来？

* 写简单的代码, 明显看不到 bug。
* 可测试性的代码, 完善的单元测试、集成测试 (AI 辅助)。
* 详细的上线 checklist。比如看数据，看日志，看压力。

> 完成的定义: 写完代码不是 "完成", 上线 checklist 看数据、看日志、看压力是基本要求。
