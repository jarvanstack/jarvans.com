# 封底估算

> 核心观点：精确数据缺失时，先用数量级估算判断方案是否可能成立，不要交白卷。

## 建议结构

1. 写一个未经估算就开始设计，最后发现数量级错误的案例。
2. 说明估算的目标是排除明显错误，不是预测精确答案。
3. 拆出输入量、增长率、峰值、单次成本和安全余量。
4. 写出假设、公式和结果范围，再用压测或监控校准。
5. 说明什么情况下必须停止估算，改做实验。
