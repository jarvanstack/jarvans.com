# 多个方案，选择最优

> 核心观点：重要问题不要只有一个方案；比较至少两个可行方案，才能看见真正的取舍。

## 建议结构

1. 写一个“第一个想到的方案并不是最好方案”的例子。
2. 说明何时值得做多方案，何时直接实现更经济。
3. 固定比较维度：收益、成本、风险、可逆性、维护负担。
4. 展示方案表和最终决策，而不只展示胜出方案。
5. 记录被放弃方案在什么条件下会变成更优解。
