# 应该确保 QA 找不出任何 bug

## 背景

我看到有些程序员会把 QA 当啄木鸟看，他们把没有把握的代码发给 QA，希望他们找到 bug。

甚至有一些团队通过 bug 数量来奖励测试人员，找到的 bug 越多，奖金越高。

我认为这是不对的，因为这样做的话，人力成本和沟通成本非常高。

没有 bug 的代码成本是最低的，其次是开发阶段发现的 bug，再其次是测试阶段发现的 bug，最严重的是上线给用户的时候发现的 bug。

## 医生

**所以应该尽量确保 QA 找不到任何 bug。**

因为这是最高效的

但大部分程序员会说: 这是不可能的。软件这么复杂，怎么可能没有 bug？

当然，他们说的没错。软件确实很复杂，不可能没有 bug。

> **人体太复杂了，不可能了解每个细节。但医生仍要尽量不出问题。**

他们不会拿人体太复杂来开脱

有的人会说: 不可能的, 你见过没有 bug 的软件吗?

你说得对，我没有见过没有 bug 的软件，但是我来这里不是跟你抬杠的

而是这是一种专业精神。

> **医生知道他的手术成功率不可能是百分之百，但是他会尽量不出问题, 是医生的专业精神。**

## 取法其上，得其中。

写出没有 bug 的代码也是程序员的专业精神。

虽然你不可能永远不失误，但是你有责任让它无限接近于零。

这种高要求的心态，往往会有 **取法其上, 得其中** 的效果。

随着不断的复盘积累经验，**你会发现你的代码质量提升比同行人快很多**。
