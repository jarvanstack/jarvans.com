# 如何正确提问？

> 核心观点：好问题让回答者快速看见目标、上下文、已知证据和真正的阻塞点。

## 提问模板

```text
目标：我想完成什么。
现状：现在发生了什么，预期是什么。
尝试：我做过哪些验证，结果如何。
判断：我目前认为原因是什么。
问题：希望对方帮助确认哪一点。
```

## 建议结构

1. 用同一个问题展示“低质量提问”和“高质量提问”。
2. 解释为什么贴一堆日志不等于提供上下文。
3. 说明什么时候给结论，什么时候只提供事实。
4. 区分开放讨论、故障求助和方案评审三种提问。
