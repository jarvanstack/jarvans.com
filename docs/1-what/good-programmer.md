# 什么是好的程序员？

> 核心观点：好的程序员不只是写代码快，而是能持续交付正确的价值，并对结果负责。

## 这篇文章要回答什么

- 好程序员与“技术熟练的人”有什么区别？
- 专业能力、业务价值和价值观分别起什么作用？
- 应该用哪些可观察的行为评价一个程序员？

## 建议结构

1. 从一个“代码写得好，但项目失败了”的真实案例开场。
2. 给出你的定义：能力决定下限，价值判断和价值观决定上限。
3. 用三个具体行为支撑定义：识别问题、做出取舍、对结果负责。
4. 说明边界：好程序员没有统一风格，但必须产生可验证的价值。
5. 用一份简短的自检问题收尾。

## 待补材料

- [ ] 一个你亲历的正面案例和反面案例
- [ ] “工程师五个等级”的来源或你的改写版本
- [ ] 评价“好”的时间尺度：一次任务、一个季度还是长期
- [ ] 可观察的行为，避免只写抽象形容词

## 可以直接写的结尾

不要问“我写了多少代码”，先问“我解决了什么问题，结果是否真的变好”。
