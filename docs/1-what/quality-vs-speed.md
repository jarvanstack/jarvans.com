# 作为程序员，我们应该更关注代码质量还是开发速度？

![代码质量 vs 开发速度](https://cdn.jarvans.com/blog/202320260713125120772.png)



我一般会从 2 个角度去观察和评价一个程序员的能力。

* 第一个是产品想要看到的：开发快，bug 少。
* 第二是技术想要实现的：高性能和可维护性。


## 2. 这4个都想要，不可能的


很明显，这是一个需要权衡的问题，因为不可能，四边形因为开发快、bug 少，性能和可维护性不能同时拉满。

你的目标就是在有限的资源下面找到最优解。

<br/>


**就像一个好的工程师，他知道现在的情况是如何，现在有多少资源，我们应该造一个快速过河的浮桥，还是坚固一些的石桥，还是能用百年的海湾大桥。**

<br/>

我见过很多程序员，他们会把海湾大桥修成浮桥，也会把浮桥当做海湾大桥慢慢修。

<br/>


## 3. 你是先提起裤子还是先跑？



我经常举一个有趣的例子：你在森林里面上厕所，旁边出现了一头熊，你是先穿裤子还是先跑？

你不跑，你就会被熊吃掉，所以你得先跑。你边跑还得边看一下熊有没有跟上来，能不能有机会把裤子穿上，因为森林里面有很多茅草，你不穿裤子会割到蛋蛋。


## 4. 最后


要做到这个，仅仅会技术是不够的，要知道公司的状况。
